## 1. 产品概述

InvestIQ 是一款由 AI 驱动的投资研究平台，帮助散户和专业投资者在几分钟内生成机构级股票研究报告。平台整合了 Claude Sonnet 4.6 ReAct Agent、SEC EDGAR 财务数据、实时行情、语义新闻检索与人机协同审查循环，通过对话界面提供买方研究台级别的分析深度。

**核心价值主张：** 针对任意美股标的提出投资问题，InvestIQ 自主收集数据、撰写包含六大模块的结构化研究报告、识别自身分析盲区，并让你决定是否深化研究——全程在一个界面内完成，6 分钟以内出报告。

---

## 2. 问题陈述

**对于散户投资者：** 专业级股票研究被锁在高价终端订阅（彭博、FactSet）或付费分析师报告后面，普通投资者只能基于不完整的信息做决策。

**对于时间紧张的从业者：** 即使是经验丰富的分析师，也有 60–80% 的研究时间花在数据收集上——拉财报、读新闻、建对比表——真正用于综合分析和判断的时间极为有限。

**现有工具的核心缺陷：**

| 工具类型 | 缺陷 |
|---|---|
| 通用大语言模型聊天工具 | 财务数据幻觉，无法访问真实财报 |
| 股票筛选工具 | 提供数据但缺乏叙事综合能力 |
| 研究报告聚合平台 | 静态内容，发布后不可交互，快速过时 |
| 新闻资讯应用 | 缺乏财务背景和结构化分析框架 |

目前市场上没有任何工具能将实时财务数据 + 语义新闻检索 + 竞品横向对比 + AI 综合分析整合为非机构用户可用的单一工作流。

## 3. 目标用户

### 主要用户 — 独立投资者 / 散户交易者
- 具备基础财务知识，关注特定股票
- 希望在买卖前深入了解一家公司
- 目前依赖手动阅读新闻、财报和分析师摘要
- **痛点：** 形成完整判断需要数小时；常在信息不全的情况下操作

### 次要用户 — 初级分析师 / 金融学生
- 为面试、竞赛或个人投资组合构建投资逻辑
- 需要结构化框架和数据访问渠道
- **痛点：** 缺乏机构数据工具的使用权限

### 第三类用户 — 小型基金 / 家族办公室
- 需要在深度尽调前快速做初步研究
- 团队规模小，无法全面覆盖多个标的
- **痛点：** 研究带宽是核心瓶颈

## 4. 目标与成功指标

| 目标 | 衡量指标 | 目标值 |
|---|---|---|
| 洞察速度 | 从提问到完整报告的时间 | 标准模式 < 3 分钟，深度模式 < 6 分钟 |

| AI 质量 | 反思审查通过率（无需补充研究） | > 60% 的报告一次通过 |

| 用户留存 | 7 天内回访率 | > 40% |
| 新用户引导 | 从安装到生成第一份报告的时间 | < 10 分钟 |

## 5. 功能需求

### 5.1 AI 研究报告生成 [P0]

**功能描述：** 用户输入自由格式的投资问题，或选择预设问题。AI Agent 自主规划并执行 8–12 次工具调用，覆盖财务数据、新闻和竞品数据源，最终生成包含六大模块的结构化报告。

**报告必须包含的六大模块：**

| 编号 | 模块 | 要求 |
|---|---|---|
| 1 | 宏观背景 | 行业趋势与宏观环境，2–3 句话 |
| 2 | 基本面分析 | 必须包含具体数字：营收、毛利率、同比/环比增速 |
| 3 | 竞品对比 | 至少对比 2 项财务指标的归一化同行对比 |
| 4 | 市场情绪 | 近期新闻、分析师观点、重大事件 |
| 5 | 核心分歧 | 多空双方论点，各 ≥ 3 条，均有数据支撑 |
| 6 | 交易建议 | 明确方向（看多/看空/中性）、目标价区间、关键跟踪指标、止损条件 |

**预设问题（适用于任意标的）：**
- 全面投资价值分析
- 盈利能力与增长趋势
- 竞品格局与相对优势
- 近期重大事件与影响
- 宏观与政策风险分析

**验收标准：**
- Agent 每份报告至少执行 6 次工具调用
- 全部 6 个模块均有内容输出
- 每个结论引用具体数据点或新闻来源
- 报告在 Streamlit 中正确渲染为 Markdown

---

### 5.2 深度分析模式 — 人机协同 [P0]

**功能描述：** 初稿生成后，第二个 LLM 实例作为"研究主管"审查报告质量。系统暂停并向用户展示审查发现，由用户决定是否授权进行补充研究。

**状态机：**

```
idle（空闲）
 └── [用户点击开始分析]
running_p1（第一轮：数据收集 + 生成初稿）
 └── [反思 LLM 审查初稿]
     ├── 通过 → 保存初稿 → done（完成）
     └── 需补充 → awaiting_approval（等待用户确认）
         ├── [用户选择：继续深化研究] → running_p2（第二轮）
         │    └── Agent 补充研究（最多 4 次工具调用）→ 保存最终报告 → done
         └── [用户选择：直接使用初稿] → 保存初稿 → done
```

**反思审查项目：**
1. 关键结论是否有具体数据支撑？
2. 竞品对比是否完整？
3. 多空论点是否平衡？
4. 交易建议是否明确？

**检查点展示内容：**
- 审查裁决（通过 / 需补充）
- 最多 2 条具体缺失点描述
- 建议的补充搜索关键词

**验收标准：**
- 检查点必须在第二轮开始前出现，用户不可绕过
- 第二轮最多执行 4 次工具调用
- 反思结果以醒目 Banner 形式在生成完成后展示（而非隐藏在日志中）
- 初稿和最终报告均保存至数据库

---

### 5.3 实时行情数据 [P0]

**功能描述：** 在市场概览页面和侧边栏展示实时价格信息。

**展示字段：** 当前价格、今日涨跌幅 %、市盈率（TTM）、预期市盈率、市值、52 周价格区间

**技术规格：**
- 数据来源：Yahoo Finance（`yfinance` 库）
- 缓存时效：5 分钟（`@st.cache_data(ttl=300)`）
- 侧边栏迷你价格卡：显示价格 + 涨跌幅（绿色/红色色码）
- 降级处理：yfinance 不可用时显示"数据暂不可用"，不阻断 AI 分析功能

**验收标准：**
- 首次加载数据在 3 秒内完成
- 5 分钟内缓存防止重复 API 请求
- yfinance 返回空数据时应用不崩溃、不卡死

---

### 5.4 动态标的支持 [P0]

**功能描述：** 用户可分析任意美股标的，不限于固定列表。

**预设标的（侧边栏下拉）：** NVDA、AMD、INTC、MSFT、TSLA、META、AAPL、GOOGL、AMZN  
**自定义输入：** 支持输入任意有效的 SEC EDGAR 股票代码  
**竞品自动映射：** 按标的预设同行；未知标的默认使用 MSFT + GOOGL

**验收标准：**
- 切换标的后，所有页面（图表、新闻、报告）同步更新
- 切换标的无需重启应用
- 无效标的在数据拉取阶段给出明确报错，而非在报告生成时才失败

---

### 5.5 新闻情报中心 [P1]

**功能描述：** 独立新闻页面，包含三个视图标签页。

**标签页一 — 新闻列表**
- 语义搜索输入框 → 对 Chroma 向量库执行余弦相似度检索
- 结果显示：标题、日期、来源、相关性得分 %
- 按日期降序排列（最新在前）

**标签页二 — 话题聚类**
- 预定义话题桶：产品与技术 / 财务表现 / 竞争格局 / 地缘政治 / 估值 / AI 需求
- 每个桶通过定向语义查询自动填充内容
- 每个话题展示最相关的 3 篇文章

**标签页三 — 市场情绪**
- 输入：最近 30 条新闻标题
- LLM 输出：看多% / 看空% / 中性% 比例、各方前 3 大信号、1 个"重点关注事件"

**验收标准：**
- 语义搜索在 2 秒内返回结果
- 情绪分析每个标的会话执行一次（切换标签页不重新触发）
- 所有标签页均遵循当前选中的标的上下文

---

### 5.6 竞品横向对比 [P1]

**功能描述：** 实时从 SEC EDGAR 拉取竞品财务数据，以归一化雷达图可视化呈现。

**对比指标：** 营收规模（相对值）、毛利率 %、净利率 %  
**数据来源：** 渲染时实时调用 EDGAR XBRL API  
**可视化：** Plotly 雷达图 + 关键洞察文字（例："NVDA 毛利率比 AMD 高 18 ppt"）

**验收标准：**
- 雷达图渲染至少涵盖 3 个标的（主标的 + 2 个竞品）
- 数据拉取期间显示加载动画
- 某竞品请求失败时跳过该标的并给出提示，不影响其他数据展示

---

### 5.7 历史报告管理 [P1]

**功能描述：** 持久化存储所有生成的报告，支持按标的浏览。

**每份报告存储内容：** 标的代码、问题、完整 Markdown 正文、完整工具调用链（工具名 + 输入参数）、时间戳、报告 ID  
**界面：** 按标的筛选的标签页 → 报告列表（日期 + 问题）→ 完整报告查看器 + 可展开的工具调用日志

**验收标准：**
- 报告在生成完成后 1 秒内保存至数据库
- 工具调用链以可读格式显示工具名和关键参数
- 历史记录跨浏览器会话持久保存（PostgreSQL 存储）

---

### 5.8 应用内设置向导 [P1]

**功能描述：** 首次使用时，用户看到引导式三步向导，无需执行任何命令行操作。

**第一步 — API 密钥配置：** 应用内表单自动写入 `.env` 文件。字段：ANTHROPIC_API_KEY（必填）、数据库主机/端口/名称/用户名/密码（必填）、NEWS_API_KEY（可选，明确标注）。

**第二步 — 数据库初始化：** 单击按钮创建三张 PostgreSQL 数据表（`filings`、`factors`、`reports`）。若 Postgres 未运行，显示明确报错并附故障排查提示。

**第三步 — 数据拉取：** 标的下拉选择 + "⚡ 一键初始化"按钮，自动拉取最近 6 个季度 SEC 财报和 25 天新闻。显示进度条。完成后出现"进入 InvestIQ"按钮。

**触发条件：**
- 环境变量中缺少 `ANTHROPIC_API_KEY`
- PostgreSQL 数据表不存在
- 用户点击侧边栏"初始化新标的"按钮

**验收标准：**
- 向导在任何页面渲染之前拦截应用启动
- 保存后密钥不在 UI 中明文显示
- 数据库初始化为每张成功创建的表显示绿色勾选
- 向导未完成时应用无法进入主界面（使用 `st.stop()` 阻断）

## 7. 技术架构

### AI 层

| 组件 | 规格 |
|---|---|
| 模型 | Claude Sonnet 4.6（`claude-sonnet-4-6`） |
| Agent 模式 | `agent_chat()` — ReAct 循环，最多 10 步 |
| 审查模式 | `simple_chat()` — 单轮调用，用于反思审查和情绪分析 |
| 重试策略 | HTTP 529（服务过载）触发，最多重试 3 次，指数退避 |

### 工具注册表

| 工具名称 | 数据来源 | 用途 |
|---|---|---|
| `query_sec_filing` | SEC EDGAR XBRL API → PostgreSQL | 原始季报数据查询 |
| `get_financial_factors` | PostgreSQL（`factors` 表） | 结构化营收、毛利率、增速指标 |
| `compare_quarterly_factors` | PostgreSQL | 季度环比与同比趋势分析 |
| `get_recent_headlines` | Chroma 向量库 | 快速浏览近期新闻 |
| `search_news` | Chroma（余弦相似度） | 特定话题语义深挖 |
| `compare_with_competitors` | SEC EDGAR XBRL API（实时） | 同行财务基准对比 |

### 数据层

| 存储 | 技术 | 内容 |
|---|---|---|
| 关系型数据库 | PostgreSQL | `filings`（原始 XBRL）、`factors`（结构化指标）、`reports`（报告 + 工具调用日志） |
| 向量数据库 | Chroma | 新闻文章向量化存储，余弦距离检索，按标的元数据过滤 |
| 市场数据 | yfinance（Yahoo Finance） | 实时价格、PE、市值、52 周区间，5 分钟缓存 |

### 前端

- 框架：Streamlit 1.32+
- 图表：Plotly（柱状图、折线叠加图、雷达图）
- 导航：`streamlit-option-menu`（水平导航栏），`st.radio` 作为降级回退
- 部署目标：本地运行（当前），架构支持单服务器云端部署

## 9. 已知限制与约束

| 约束 | 用户影响 | 当前应对措施 |
|---|---|---|
| NewsAPI 免费额度：仅限最近 30 天，每日 100 次请求 | 较早历史事件的新闻覆盖有限 | 向导中明确标注为可选；无 API Key 时财报分析仍可正常使用 |
| SEC EDGAR 竞品实时请求：延迟 5–15 秒 | 雷达图加载较慢 | 加载期间显示 Spinner |
| 依赖本地 PostgreSQL | 安装有一定门槛 | 设置向导引导完成；未来可迁移至 SQLite 实现零配置 |
| 无 PDF 导出功能 | 报告无法离线分享 | 列入未来路线图 |
| Mac 开发环境存在两个 Conda 环境 | 可能出现包导入失败 | 所有可选依赖均添加 `try/except ImportError` 降级处理 |
| yfinance 访问频率限制 | 高并发时行情数据可能失败 | 5 分钟缓存降低调用频率；失败时优雅降级并提示 |

## 10. 产品路线图

### 近期（下一个迭代周期）
- 每日简报 Agent：定时任务触发轻量报告（当日头条 + 较前一日的财务变化）
- 报告 PDF 导出功能
- SEC 8-K 事件摄取作为主要事件数据源（移除对 NewsAPI 的依赖，免费，实时）

### 中期
- 多 Agent 架构：财务 Agent / 新闻 Agent / 综合 Agent，配合显式协调器
- 扩展估值指标：P/S、EV/EBITDA（通过 yfinance 获取）
- 财报日历集成（在报告中突出显示下次财报时间）
- Agent 可观测性视图：工具调用链可视化 DAG，每步延迟，标注哪些调用来自反思补充轮

### 长期
- 用户账号体系与报告分享功能
